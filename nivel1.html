Contexto· El problema de la carpeta de cargas

El archivo no tan inofensivo

El escenario plantea un portal universitario habilitado para que los estudiantes carguen tareas y documentos. Tras una actualización del sistema, se detectaron archivos almacenados que comprometían la estabilidad del servidor, generando sospechas de que usuarios externos habían cargado archivos no autorizados.

El código problemático:

// Este código parecía inofensivo, pero era muy peligroso
String fileName = request.getParameter("fileName");
// obtiene el nombre del archivo enviado por el formulario
File file = new File("/uploads/" + fileName);
// crea la ruta de destino usando el nombre del usuario sin validar
FileOutputStream fos = new FileOutputStream(file);
// abre un flujo para escribir el archivo en el servidor
fos.write(request.getInputStream().readAllBytes());
// copia los bytes enviados por el usuario al archivo de destino
fos.close();
// cierra el flujo de salida para finalizar la escritura
out.println("Archivo cargado");
// responde al usuario con un mensaje de confirmación

¿Ves algo raro?

Este código:

  • Toma el nombre del archivo directamente de lo que el usuario escribe
  • No verifica qué tipo de archivo es
  • No controla el tamaño del archivo
  • No pregunta quién está subiendo el archivo

¿Qué salió mal?

Al confiar en cualquier cosa que el usuario le dé, el código permite que un atacante "le mienta" al servidor. Un atacante no va a subir area.pdf, va a subir algo como ..//..//etc/passwd o un archivo virus.php

Consecuencias: Archivos maliciosos, sistema lento, datos expuestos, portal cerrado por días.
