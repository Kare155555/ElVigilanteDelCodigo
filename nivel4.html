SAST

  • Qué es
    SAST es una herramienta de seguridad de aplicaciones (AppSec) que escanea el código fuente, binario o de bytes de una aplicación. Se trata de una herramienta de pruebas de caja blanca que identifica la causa raíz de las vulnerabilidades.
  • Cómo funciona
    Realiza un análisis estático del código para revisar fallos de codificación y diseño antes de que la aplicación se ejecute.
  • Importancia
    Detecta vulnerabilidades críticas antes del despliegue, cuando aún son más baratas de corregir. Si se integra en CI/CD, se habla de DevSecOps.
  • Ventajas
    Escanea el código fuente, ofrece informes en tiempo real, cubre múltiples lenguajes y ayuda a educar a los desarrolladores.
  • Contras
    No identifica vulnerabilidades externas, puede producir falsos positivos y requiere acceso al código fuente.
  • Ejemplo
    SAST podría detectar una línea como String fileName = request.getParameter("fileName") y alertar sobre entrada no validada y Path Traversal.

DAST

  • Qué es
    DAST es el análisis dinámico de una aplicación web desde el front-end para encontrar vulnerabilidades mediante ataques simulados.
  • Cómo funciona
    Escanea una aplicación en ejecución y compara el comportamiento real con los patrones de ataque esperados.
  • Importancia
    Detecta fallos que aparecen solo cuando la app está corriendo, como errores de configuración o validación.
  • Ventajas
    No requiere acceso al código fuente, valida el entorno real y puede descubrir vulnerabilidades explotables.
  • Contras
    No indica la línea exacta del fallo, requiere experiencia para interpretar informes y puede ser lento.
  • Ejemplo
    DAST podría enviar ../../etc/passwd en el campo de archivo y confirmar si el servidor responde con contenido sensible.

Manual

  • Qué son
    Las pruebas manuales consisten en ejecutar casos de prueba directamente por una persona, sin automatización, simulando el comportamiento del usuario final.
  • Ciclo
    Revisión de requisitos, diseño de casos, ejecución, registro de resultados, reporte de bugs y reejecución tras correcciones.
  • Habilidades
    Atención al detalle, pensamiento crítico, curiosidad exploratoria y comunicación efectiva.
  • Cuándo usar
    En primeras etapas del desarrollo, validación visual y UX, pruebas exploratorias y UAT con usuarios reales.
  • Valor
    El tester humano puede encontrar errores lógicos y de experiencia que las herramientas automatizadas suelen pasar por alto.
