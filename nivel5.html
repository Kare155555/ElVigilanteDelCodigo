SonarQube

  • Qué es SonarQube es una herramienta de seguridad y calidad de código que ayuda a los desarrolladores a identificar errores, malas prácticas de programación, duplicaciones y vulnerabilidades en sus aplicaciones. Es open source y compatible con más de 27 lenguajes de programación.

  • Cómo funciona SonarQube está dividido en dos partes: el servidor, donde se encuentra el motor de computación que realiza el análisis y almacena los resultados, y el SonarQube Scanner, que se ejecuta sobre el código.

  • Por qué usarlo Detecta errores y vulnerabilidades, mejora la mantenibilidad, aplica estándares de codificación, integra con IDEs mediante SonarLint y funciona en pipelines de CI/CD.

  • Métricas Calcula código duplicado, cobertura, code smells, bugs, vulnerabilidades y deuda técnica basada en SQALE.

  • VS Code En VS Code se instala SonarLint desde Extensiones, se reinicia y luego se conecta con SonarQube cuando sea necesario para resaltar problemas mientras se escribe código.

  • Comandos Para proyectos Maven se usa mvn sonar:sonar. Con GitLab CI/CD se puede usar mvn org.sonarsource.scanner.maven:sonar-maven-plugin:sonar -Dsonar.qualitygate.wait=true

OWASP ZAP

  • Qué es OWASP ZAP (Zed Attack Proxy) es una herramienta de pruebas de penetración que ayuda a desarrolladores y profesionales de la seguridad a detectar y encontrar vulnerabilidades en aplicaciones web. Es open source y la #1 recomendada por OWASP.

  • Cómo funciona ZAP se sitúa entre una aplicación web y un cliente de pruebas de penetración. Funciona como un proxy, capturando los datos transmitidos y determinando cómo responde la aplicación a posibles solicitudes maliciosas.

  • Capacidades Proxy interceptor, Spider, escáner activo, escáner pasivo, fuzzer e integración CI/CD.

  • Escaneo El escaneo pasivo analiza solicitudes y respuestas sin modificarlas, mientras que el activo crea y modifica solicitudes para enviar pruebas de ataque.

  • Spider El Spider tradicional inspecciona HTML en busca de enlaces; el Spider AJAX renderiza JavaScript y sigue enlaces en aplicaciones con JavaScript.

  • Autenticación ZAP soporta autenticación basada en formularios, scripts, JSON y HTTP/NTLM.

  • Comandos Se puede abrir con zap.sh, en modo headless con zap.sh -daemon -port 8080, o con comandos CLI como zap-cli quick-scan --self-contained http://target.com
